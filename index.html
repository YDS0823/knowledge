线上请求超时问题排查记录

2024-03-15 · 后端服务 · Node.js

问题描述

生产环境偶发请求超时，错误日志显示 ETIMEDOUT， 平均每小时出现 3~5 次，集中在业务高峰期。

排查过程

首先查看 Nginx 访问日志，确认超时请求的特征：

grep 'upstream timed out' /var/log/nginx/error.log | tail -20

发现超时均指向同一个下游服务，怀疑是连接池耗尽。检查连接池配置：

// config.js
pool: {
  max: 10,       // 最大连接数
  min: 2,
  acquire: 3000, // 获取连接超时时间(ms)
  idle: 10000
}

高峰期并发量约 80 QPS，而连接池上限仅为 10， 导致请求在等待连接时超过 acquire 阈值而报错。

解决方案

将连接池 max 调整为 50，同时增加重试逻辑，上线后超时问题消失。

后续优化

考虑引入熔断机制，避免下游异常时连接池被打满， 同时对慢查询 SQL 做索引优化以降低单次请求耗时。
